# 工具网站前端架构设计

## 1. 推荐技术栈说明

### 主要技术栈
- **HTML5**: 用于构建页面结构和语义化标记
- **Tailwind CSS**: 用于快速构建响应式界面，提供一致的设计语言
- **原生 JavaScript**: 实现交互逻辑和工具功能，确保高性能和兼容性
- **GitHub Pages**: 静态网站托管平台

### 备选技术栈
- **Vue.js 3**: 如需更复杂的组件化和状态管理，可选择轻量级的 Vue.js
- **Vite**: 作为开发服务器和构建工具，提供快速的开发体验

### 工具链
- **npm / yarn**: 包管理工具
- **PostCSS**: CSS 处理工具，配合 Tailwind CSS 使用
- **Autoprefixer**: 自动添加浏览器前缀
- **ESLint + Prettier**: 代码规范和格式化工具
- **Husky + lint-staged**: Git 钩子工具，确保代码质量

### 开发环境
- **VS Code**: 推荐编辑器，有丰富的插件生态
- **Live Server**: 本地开发服务器插件
- **Tailwind CSS IntelliSense**: Tailwind CSS 语法提示插件

## 3. 文件职责说明

### 核心配置文件
- **package.json**: 项目元数据、依赖声明和脚本命令
- **tailwind.config.js**: Tailwind CSS 配置文件，包括主题定制和插件配置
- **postcss.config.js**: PostCSS 配置，集成 Tailwind CSS 和 Autoprefixer
- **vite.config.js**: Vite 构建工具配置（如选择 Vue 方案）
- **.gitignore**: 定义 Git 版本控制忽略的文件和目录

### 源代码目录 (src/)
- **main.js**: 应用程序入口文件，初始化应用和路由
- **App.vue**: 根组件，包含公共布局结构
- **components/**: 存放可复用的 UI 组件
- **pages/**: 存放页面级别组件，每个工具作为一个独立页面
- **styles/**: 全局样式文件和 Tailwind CSS 配置
- **utils/**: 工具函数和辅助方法
- **router/**: 路由配置，定义页面访问路径

### 静态资源目录 (public/)
- 存放不需要经过构建过程的静态文件
- 直接映射到网站根路径下
- 包括 favicon、robots.txt 等 SEO 相关文件

### 构建输出目录 (dist/)
- 存放构建后的生产环境静态文件
- 用于 GitHub Pages 部署

## 5. 版本管理和 Commit 规范

### 分支策略
- **main**: 主分支，稳定版本，用于 GitHub Pages 部署
- **develop**: 开发分支，日常开发在此进行
- **feature/***: 功能分支，新功能开发使用
- **hotfix/***: 热修复分支，紧急 bug 修复使用

### Commit 规范 (Conventional Commits)
采用 Conventional Commits 规范，格式如下：

```
<type>[optional scope]: <description>

[optional body]

[optional footer(s)]
```

#### Commit 类型
- **feat**: 新功能
- **fix**: 修复 bug
- **chore**: 构建工具或辅助工具的变动
- **ci**: CI 配置相关
- **docs**: 文档更新
- **style**: 代码格式调整（不影响代码运行）
- **refactor**: 重构代码
- **perf**: 性能优化
- **test**: 测试相关代码

#### 示例
```
feat(tools): 添加 JSON 校验工具

- 实现 JSON 格式校验功能
- 添加格式化和压缩选项
- 增加错误提示和行号定位

Closes #123
```

```
fix(search): 修复搜索功能无法匹配中文的问题

- 更新搜索算法以支持中文分词
- 优化模糊匹配准确率
- 添加搜索历史记录功能

Fixes #456
```

### 发布规范
- 使用语义化版本控制（SemVer）：MAJOR.MINOR.PATCH
- 每次发布前进行 CHANGELOG 更新
- 通过 GitHub Releases 记录版本变更

### Pull Request 规范
- PR 标题遵循 Conventional Commits 格式
- 描述中包含变更内容和影响范围
- 关联相关 Issue
- 通过 CI 检查和代码审查后方可合并

## 6. 部署方案

### GitHub Actions 自动部署
- 监听 main 分支 push 事件
- 自动构建项目并部署到 GitHub Pages
- 构建前运行测试和代码检查
- 部署成功后发送通知

### 环境变量配置
- 使用 GitHub Secrets 管理敏感信息
- 通过环境变量区分开发和生产环境配置

## 7. 性能优化建议

### 构建优化
- 代码分割，按需加载页面组件
- 静态资源压缩和优化
- 利用浏览器缓存策略

### 运行时优化
- 减少 DOM 操作
- 使用防抖和节流优化高频事件
- 懒加载非关键资源

### SEO 优化
- 合理使用语义化 HTML 标签
- 为每个工具页面添加 meta 描述
- 提供站点地图 sitemap.xml
