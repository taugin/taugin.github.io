# 工具类网站静态站点需求文档

## 1. 项目定位

本项目是一个基于 GitHub Pages 的纯前端工具箱网站，旨在为用户提供一系列实用的小工具，包括文本处理、数据校验、格式转换等功能。网站将采用简洁直观的设计风格，确保用户能够快速找到并使用所需工具，所有操作均在浏览器端完成，无需后端支持。

### 目标
- 提供一个轻量级、快速响应的在线工具集合平台
- 实现完全静态部署，利用 GitHub Pages 进行免费托管
- 支持移动端和桌面端访问，具有良好的响应式设计
- 保证用户数据隐私，所有处理过程都在本地浏览器中进行

## 2. 用户场景

### 主要用户角色
1. **开发人员**
   - 需要频繁进行 JSON 格式校验、编码解码、时间戳转换等操作
   - 希望有快捷的在线工具替代本地安装的软件

2. **数据分析师**
   - 需要对文本数据进行清洗、格式化处理
   - 经常需要验证数据格式正确性

3. **普通办公人员**
   - 偶尔需要进行文本大小写转换、去除空格等简单操作
   - 寻求便捷易用的在线工具替代复杂软件

### 典型使用场景
1. 开发者在编写代码时需要快速验证一段 JSON 是否格式正确
2. 数据分析师从不同来源获取的数据需要统一格式处理
3. 办公人员需要批量处理文本内容，如去除多余空格、转换大小写等
4. 用户需要临时进行 Base64 编码或解码操作
5. 需要快速查询 Unix 时间戳对应的具体时间
6. 开发者需要测试API接口，但不想使用复杂的Postman等工具
7. 前端开发需要快速构造HTTP请求来测试CORS或API响应

## 6. 技术选型建议

### 前端技术栈
- HTML5 + CSS3 + JavaScript (原生实现，避免过多依赖)
- 可考虑轻量级框架如 Vue.js（便于组件化开发）
- Bootstrap 或 Tailwind CSS（用于快速构建响应式界面）
- Clipboard.js（用于复制到剪贴板功能）

### 构建与部署
- 使用 GitHub Actions 自动部署到 GitHub Pages
- 静态资源优化（图片压缩、代码压缩）
- PWA 支持（可选，提升用户体验）

### 开发工具
- VSCode 作为主要编辑器
- Git 版本控制
- ESLint 代码规范检查

### 第三方库（仅限纯前端）
- moment.js（时间处理）
- crypto-js（加密算法）
- prism.js（代码高亮）

### HTTP请求相关特殊考虑
- 使用浏览器原生 fetch API 或 XMLHttpRequest
- 需要明确告知用户CORS限制以及如何解决
- 对于跨域请求，可能需要用户自行设置服务器端CORS头或使用代理

## 8. HTTP/HTTPS 模拟请求工具详细规格

### 功能需求
1. **基本请求功能**
   - 支持 GET、POST、PUT、DELETE、PATCH、HEAD、OPTIONS 等常见HTTP方法
   - URL 输入验证（必须是有效的 http:// 或 https:// 地址）
   - 可自定义请求头（添加、编辑、删除键值对）
   - 请求体编辑支持多种格式：
     - 纯文本
     - JSON（自动格式化和校验）
     - Form 表单数据
     - XML

2. **响应处理**
   - 显示完整的响应信息：
     - 状态码和状态文本
     - 响应头信息（原始格式展示）
     - 响应体内容（支持语法高亮）
   - 根据 Content-Type 自动选择合适的展示方式
   - 响应时间统计

3. **用户体验**
   - 请求历史记录（本地存储）
   - 支持导出/导入请求配置为JSON格式
   - 快速填充常用API端点
   - 快捷键支持（如 Ctrl+Enter 发送请求）

4. **安全与限制**
   - 明确提示CORS相关的限制
   - 不在任何服务器上记录用户请求
   - 提供使用代理解决跨域问题的说明文档
   - 请求超时设置（默认10秒）

### 界面设计要点
- 清晰区分请求配置区和响应结果显示区
- 请求方法使用醒目的颜色标识
- 错误信息友好提示（如网络错误、CORS限制等）
- 响应体过大时提供截断或分页显示
- 支持深色模式以适应不同用户偏好
